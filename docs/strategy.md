# אסטרטגיה, Positioning, ארכיטקטורה, Roadmap — ברידג' גליל

## 1. Executive summary
שלושת המתחרים בעברית חלשים בדיוק במקומות שקל לנצח בהם: אפס SEO בסיסי (אין H1/meta), אפס תרגול אינטראקטיבי בתוך האתר (מיקור חוץ ל-BBO), אפס נוכחות בצפון, ואפס UX לקהל 55+. נבנה אתר סטטי מהיר Hebrew-first עם תרגול בדפדפן, משפך "תוכן חינמי → מפגש ראשון חינם → קורס", ו-roadmap מנועים מאומת (WASM DD solver ברישיון Apache). הפרוטוטיפ בנוי, נבדק, ומוכן לפריסה.

## 2. Positioning
"המורה שילמד אתכם ברידג' נכון, בעברית, בגובה העיניים."
- אנושי וחם, לא פלטפורמה אנונימית (מול BOA) ולא פורטל עמוס (מול GoBridge).
- מקצועי ויסודי: שיטה מדורגת, כל שיעור בנוי על קודמו.
- מקומי-חזק: היחיד בגליל המערבי, עם זרוע אונליין ארצית.
- טכנולוגי בשקט: התרגול החכם הוא הוכחה, לא סיסמה.
אנטי-מטרות: לא וורדפרסי, לא קזינו, לא עמוס, לא "אתר AI גנרי".

## 3. Product strategy — 4 שכבות
1. אמון/תדמית: דף בית פרימיום, אודות אישי, המלצות, עיצוב מועדון-יוקרתי.
2. ערך חינמי: מרכז ידע (שיעורים+מאמרים), מילון מונחים, תרגול חופשי, אתגר יומי → טראפיק SEO ואמון.
3. למידה מונחית: 3 קורסים מדורגים + זום + מפגש ראשון חינם. מחיר פרימיום מעל תקרת ה-280₪ של Salon, מוצדק בקבוצות קטנות.
4. בידול טכנולוגי: חידונים גנרטיביים היום; DD solver בדפדפן (V2); AI הכרזות (V3). ראה research-engines.md.

## 4. Content + SEO strategy
- Pillar: מרכז הידע. Cluster: שיעורי יסוד → מאמרי מוסכמות → מילון מונחים (DefinedTermSet schema).
- Local SEO: עמודי עיר (עכו, נהריה, כרמיאל, חיפה, גליל מערבי) עם LocalBusiness schema, תוכן ייחודי לכל עמוד, קישור פנימי לקורסים. אף מתחרה לא עושה זאת → ניצחון קל ב"ברידג' בנהריה" וכו'.
- מילות מפתח ראשיות: קורס ברידג' למתחילים, לימוד ברידג', ברידג' אונליין, ברידג' בעברית, ברידג' + [עיר].
- טכני: H1 יחיד, meta ייחודי, JSON-LD (LocalBusiness/Course/DefinedTermSet), sitemap.xml, טעינה <1s, אפס JS חוסם.
- קצב מומלץ לנפתלי: מאמר אחד בשבועיים + עדכון האתגר היומי אוטומטי (כבר גנרטיבי).

## 5. Tech architecture
### עכשיו (V1, בנוי)
Static HTML/CSS/vanilla JS. אפס backend, אפס build. bridge.js (מודל קלפים) + quiz.js (מנוע חידונים). Hosting: Cloudflare Pages/Netlify (חינם, CDN, מהיר בישראל). Forms: WhatsApp deep-link + mailto (מתאים לקהל; אפס תחזוקה).
### V2
- Frontend: נשאר סטטי; אם התוכן גדל → Astro (content collections, RTL friendly, static output).
- Engine בדפדפן: vendored `bridge-dds` WASM ב-Web Worker.
- תוכן ידיים: חבילות JSON מחוללות offline עם endplay.
- Search: Pagefind (סטטי, עברית OK). Analytics: Plausible (בלי באנר עוגיות). CMS: markdown בגיט; אם נפתלי רוצה עריכה עצמית → Decap CMS (git-backed, חינם).
### V2.5-V3
- Engine API: FastAPI + endplay/libdds על Fly.io/VPS קטן; cache DD לפי hash חלוקה (SQLite מספיק).
- ben כשירות Docker GPL מבודד.
- Auth+DB (רק כשיש צורך אמיתי במעקב התקדמות): Supabase (Postgres+auth+RLS).
- Observability: Sentry frontend + uptime ping. Jobs: cron פשוט לחילול חבילות ידיים.
עיקרון: לא מוסיפים שכבה לפני שיש לה משתמשים. הסטטי נשאר הליבה כי הוא ה-SEO וה-speed advantage.

## 6. UX/UI system (rationale מול מתחרים)
- פלטה: ירוק שולחן כהה #1B4D3E + קרם #F7F3EA + זהב #C9A227. מועדון יוקרתי, לא קזינו ולא corporate. GoBridge=Wix גנרי, BOA=Elementor עמוס, Salon=ASP מיושן → כל פלטה עקבית מנצחת.
- טיפוגרפיה: Heebo/Assistant, base 18px, ניגודיות AA+. קהל 55+: כפתורים גדולים, שורות קצרות, אפס אנימציות מיותרות.
- קומפוננטות: cards עם פס זהב, course-cards עם header ירוק, hand display (LTR מבודד בתוך RTL), quiz-box עם משוב צבע ירוק/אדום.
- CTA: זהב=המרה ראשית (הרשמה/וואטסאפ), ירוק=משני (תוכן). אחד ראשי לעמוד. לא אגרסיבי: אפס טיימרים/פופאפים (בניגוד ל-BOA).
- Mobile: hamburger, grid קורס בעמודה, quiz 2 עמודות. נגישות: סמנטיקה, aria-label, ניווט מקלדת בחידונים (buttons אמיתיים).

## 8. Assumptions & tradeoffs
- מחירים 480-640₪ = placeholders מסומנים; מעל תקרת Salon בכוונה (פרימיום, קבוצות קטנות).
- טלפון/מייל/המלצות = TODO לנפתלי. אתר עולה לאוויר רק אחרי החלפתם.
- אין תשלום אונליין ב-V1: הקהל מעדיף טלפון/וואטסאפ, וזה מוריד friction ותחזוקה. Stripe/משולם רק כשיש ביקוש מוכח.
- סטטי לפני CMS: נפתלי לא כותב HTML, אבל קצב תוכן נמוך בהתחלה → עריכה דרכי היא זולה יותר מתחזוקת CMS. Decap כשיכאב.
- אתגר יומי דטרמיניסטי (seed=תאריך) במקום שרת: אפס עלות, אותה חוויה.
